## はじめに

e2eテストの実装やメンテナンスに疲れていませんか？

最近、Seleniumの開発者が手がけた **Vibium** というブラウザ自動化ツールが登場しました。  
AIエージェントとの連携を前提に設計されており、自然言語でブラウザ操作が可能です。

今回は、Claude Codeと組み合わせて、

> 自然言語ベースのe2eテストは実用的なのか？

を検証してみました。

## Vibiumとは

Vibiumは、AIエージェントによる操作を前提に設計されたブラウザ自動化ツールで、特徴は以下の通りです。

- AIが理解しやすい構造
- MCP経由でエージェント操作が可能
- JS/TS, Pythonでの利用も可能

https://github.com/VibiumDev/vibium

## Claude CodeでVibiumを利用する方法

この記事では、VibiumのMCPサーバーを利用して、Claude Codeでブラウザ操作をする方法をご紹介します。
やり方は簡単で、以下のコマンドを実行すればセットアップは完了です。

```.bash
claude mcp add vibium -- npx -y vibium mcp
```

試しに、Claude Codeに以下の指示をすると、VibiumのMCPを呼び出して、ブラウザを立ち上げて、操作をしてくれます。

```.bash
❯ ホーム画面のスクリーンショットを撮って
```

## Claude Codeでe2eテストを行う

ざっくりとしたe2eテストのシナリオのmdファイルを配置して、それを読み込ませてClaude Codeでテストをさせてみました。シナリオの順番通りに操作して、各期待する動作を確認してくれました。
![alt text](/images/20260215/image2.png)

Seleniumと比較すると、以下のようなメリット、デメリットが挙げられます。

| 比較項目     | Selenium | Vibium × Claude |
| ------------ | -------- | --------------- |
| 記述方法     | コード   | 自然言語        |
| メンテナンス | 手動修正 | AIが解釈        |
| 実行速度     | 速い     | やや遅い        |
| 柔軟性       | 高い     | 高いが曖昧      |
| 学習コスト   | 高め     | 低い            |

VibiumのMCPでテストを実行すると、ソースコードでの記載が必要なく、日本語でテストの指示ができて、テストを開始するまでのスピードが早いです。また、ソース上の修正が入っても、よしなにAIがシナリオを理解して、テストをしてくれるので、テストコードのメンテナンスが楽になる点もメリットになります。

ただ、テストの実行完了するまでの時間が長いなと感じました。指示の仕方を工夫すれば、テストの実行時間の削減ができるかもしれませんが、テストを何回も回して、修正を進めたい動き方をしたい時には不向きだと考えられます。また、期待値の確認がちゃんとできているかが分かりづらいので、大雑把な項目を確認したいテストの方が向いていそうです。

そのため、e2eテストを行う用途よりも、ブラウザ操作をして改善点を探させる用途のほうが適していると感じました。

## まとめ

AIエージェント主導のe2eテストは、従来のコードベースのテストとは別の価値を持つアプローチだと感じました。

現時点では高速な回帰テストには向きませんが、
探索的テストやUX改善の観点では非常に強力なツールになり得ます。

2026/02時点で最新のversionが0.1.8なので、今後のアップデート次第では、e2eテストの書き方そのものが変わる可能性もあると考えられます。
